# ri-tts

Text-to-speech model training using discrete audio tokens. Trains a language model (Qwen3-0.6B) to predict [DAC](https://github.com/descriptinc/descript-audio-codec) codebook tokens from text input, then decodes those tokens back to audio.

## How it works

```
Text: "Hello world"
  -> Model generates: <|audio_start|><|c1_551|><|c2_118|><|c3_42|>...<|audio_end|>
    -> DAC decoder produces: audio waveform (44.1kHz WAV)
```

The model learns to map text to sequences of discrete audio codes from DAC's first 3 residual vector quantization codebooks. No spectrograms, no vocoders — just next-token prediction on audio codes.

## Quick start

### Requirements

- Python 3.10+
- CUDA GPU (A100/H100 recommended) or Apple MPS
- ~10GB disk for model + data

### Setup

```bash
git clone https://github.com/treadon/ri-tts.git
cd ri-tts
pip install -r requirements.txt
```

### Train

```bash
./train.sh
```

The launcher handles everything:
1. Checks W&B login (optional, for experiment tracking)
2. Builds the tokenizer if needed
3. Manages a tmux session for training (resume-safe)

Training pulls data automatically from [treadon/speech-dac-tokens-3cb](https://huggingface.co/datasets/treadon/speech-dac-tokens-3cb) on HuggingFace (217K samples, ~704 hours of speech).

On an A100, expect ~4-6 hours for a full run. Training auto-detects your hardware:
- **CUDA**: bf16, batch size 4, gradient accumulation 4
- **MPS**: fp32, batch size 1, gradient accumulation 16

### Generate speech

```bash
python decode.py "Hello, how are you?" -o hello.wav
```

Or decode from saved token files (generated during training):

```bash
python decode.py --from-tokens samples/step_001000/sample_0.txt -o test.wav
```

## Dataset

[treadon/speech-dac-tokens-3cb](https://huggingface.co/datasets/treadon/speech-dac-tokens-3cb)

- 217,250 samples from LibriSpeech (clean-360 + other-500)
- ~704 hours of English speech
- 3 DAC codebooks (c1=coarse, c2=detail, c3=fine)
- Pre-tokenized and ready for training

## Architecture

| Component | Details |
|-----------|---------|
| Base model | [Qwen3-0.6B](https://huggingface.co/Qwen/Qwen3-0.6B) |
| Audio codec | [DAC 44kHz](https://github.com/descriptinc/descript-audio-codec) |
| Codebooks | 3 (of 9 available), 1,024 entries each |
| Context window | 4,096 tokens |
| Vocab | 154,743 (151,669 base + 3,074 audio tokens) |
| Training | Causal LM, cosine schedule, 3 epochs |

## Project structure

```
ri-tts/
├── train.sh              # Training launcher (wandb, tokenizer, tmux)
├── train.py              # Training script (HF Trainer, auto-detect GPU)
├── decode.py             # Inference: text -> DAC tokens -> WAV
├── build_tokenizer.py    # Extends Qwen3 tokenizer with DAC tokens
├── requirements.txt      # Python dependencies
└── tokenizer/            # Built tokenizer (generated by build_tokenizer.py)
```

## Monitoring

Training logs to [Weights & Biases](https://wandb.ai) (optional). At every checkpoint (every 500 steps), the model generates test audio tokens so you can track quality over time.
